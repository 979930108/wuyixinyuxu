
访问：http://localhost:3000

### 2.2 本地环境变量（`.env.local`）

```env
ACTIVATION_SECRET=dev-secret-key-for-local-testing-only-32chars
ACTIVATION_CODES=DEMO-2026-PRO:lifetime,TEST-7DAY-001:7,XIANYU-VIP-8888:lifetime
```

| 文件 | 作用 | 是否提交 Git |
|------|------|-------------|
| `.env.example` | 模板说明，给别人参考 | ✅ 提交 |
| `.env.local` | 本地真实配置 | ❌ 不提交（已在 .gitignore） |

> **注意**：改 `.env.local` 后需重启 `npm run dev`。本地配置**不会**影响线上。

### 2.3 本地 npm 源问题（公司电脑必读）

如果在公司电脑开发，`npm install` 可能走公司内网 Nexus（如 `nexus.mcd.com.cn`），会导致 `package-lock.json` 里全是内网地址，**Vercel 构建必失败**。

项目已添加 `.npmrc`：

```
registry=https://registry.npmjs.org/
```

若 lock 文件被污染，在本机执行：

```bash
rm -rf node_modules package-lock.json
NPM_CONFIG_REGISTRY=https://registry.npmjs.org npm install --registry=https://registry.npmjs.org/
```

然后 commit 新的 `package-lock.json` 并 push。

```bash
git add .

git push origin main
```

Push 后 Vercel **自动触发重新部署**，一般无需手动操作。

## 4. Vercel 完整配置流程（重点）

本章按实际操作顺序，从零完成 Vercel 注册、导入项目、构建配置、环境变量、域名绑定、重新部署。

### 4.0 整体流程图

```
GitHub 有代码
    ↓
Vercel 注册 / 登录
    ↓
Import 项目 wuyixinyuxu
    ↓
Framework 选 Next.js → Deploy
    ↓
（若失败）改 Build 设置 + 检查 package-lock / vercel.json
    ↓
Settings → Environment Variables 配密钥和激活码
    ↓
Settings → Domains 添加 shuzigongju.com
    ↓
阿里云 DNS 添加 A + CNAME
    ↓
Vercel Domains 点 Refresh → 变绿
    ↓
Redeploy → 线上可访问
```

### 4.1 注册与登录 Vercel

1. 打开 https://vercel.com
2. 点 **Sign Up**，建议用 **GitHub 账号**登录（方便直接读仓库）
3. 授权 Vercel 访问你的 GitHub

> 登录后左上角显示团队名（如 `wuyixinyu`），下面才是具体项目。

---

### 4.2 首次导入 GitHub 项目

#### 步骤

1. Vercel 首页点 **Add New...** → **Project**
2. 在 **Import Git Repository** 列表找到 `wuyixinyuxu`
   - 若看不到：点 **Adjust GitHub App Permissions**，给 Vercel 授权访问该仓库
3. 点 **Import**

#### Configure Project 页面（第一次部署前）

| 字段 | 填什么 | 说明 |
|------|--------|------|
| Project Name | `wuyixinyuxu` | 可保持默认 |
| Framework Preset | **Next.js** | ⚠️ 必须是 Next.js，不能选 Other |
| Root Directory | `./` 留空 | 代码在仓库根目录 |
| Build Command | 留空 | 默认 `next build` |
| Output Directory | 留空 | ⚠️ Next.js 不要填 public/dist |
| Install Command | 留空 | 默认 `npm install` |

4. **Environment Variables** 可先跳过，后面再配
5. 点 **Deploy**

#### 等待构建

- 进入 **Deployments** 页，看构建日志
- 成功：状态变 **Ready**（绿点）
- 失败：状态 **Error**，点进去看 **Building** 日志红色报错

---

### 4.3 构建与开发设置（必查）

路径（注意 URL 里要有项目名）：

```
https://vercel.com/wuyixinyu/wuyixinyuxu/settings
→ 左侧 Build and Deployment
```

> ⚠️ 旧文档叫「Build & Development Settings」，新版叫 **Build and Deployment**，在 Settings 左侧第二项。

#### Framework Settings 对照表

| 配置项 | 正确值 | 错误值及后果 |
|--------|--------|--------------|
| **Framework Preset** | `Next.js` | `Other` → 找 public 目录，构建报 404 |
| **Node.js Version** | 默认 20.x 即可 | — |
| **Build Command** | 留空（Override 关） | 乱填会导致构建失败 |
| **Output Directory** | **留空**（Override 关） | 填 `public` → No Output Directory named "public" |
| **Install Command** | 留空（Override 关） | — |
| **Development Command** | 留空 | — |

#### 如何修改

1. **Framework Preset** 下拉选 **Next.js**
2. 下面 Build Command / Output Directory 等，**Override 开关保持关闭**（灰色）
3. 页面左下角点 **Save**

#### 项目内 vercel.json（已配置，勿删）

仓库根目录 `vercel.json` 会覆盖部分设置，确保 Vercel 识别 Next.js：

```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "framework": "nextjs",
  "buildCommand": "next build"
}
```

### 4.4 环境变量配置

#### 入口（最容易找错）

```
✅ 正确（项目级）：
   vercel.com/wuyixinyu/wuyixinyuxu/settings/environment-variables

❌ 错误（团队级，没有项目变量）：
   vercel.com/wuyixinyu/~/settings/environment-variables
```

导航：**项目 wuyixinyuxu → 顶部 Settings → 左侧 Environment Variables**

#### 添加 ACTIVATION_SECRET

1. 点 **Add Environment Variable**
2. 填写：

| 字段 | 值 |
|------|-----|
| Key | `ACTIVATION_SECRET` |
| Value | 32 位以上随机字符串，如 `resume-secret-2026-Kp8mN2qR5vX9wZ3jL7hF4tY6` |
| Environments | 勾选 **Production** 和 **Preview** |
| Sensitive | **开着**（滑块在右、蓝色） |

3. 点 **Save**
4. **自己把密钥抄到备忘录**，Sensitive 保存后无法再查看

#### 添加 ACTIVATION_CODES

1. 再点 **Add Environment Variable**
2. 填写：

| 字段 | 值 |
|------|-----|
| Key | `ACTIVATION_CODES` |
| Value | `XIANYU-VIP-8888:lifetime`（或多个逗号分隔） |
| Environments | **Production** 和 **Preview** |
| Sensitive | **关着**（滑块在左、灰色）← 方便以后编辑看见 |

3. 点 **Save**

#### 激活码格式速查

```
永久会员：  XIANYU-VIP-8888:lifetime
30 天会员： XIANYU-30DAY-001:30
7 天会员：  PROMO-7DAY:7
多个码：    码1:lifetime,码2:30,码3:lifetime
```

> ⚠️ 编辑 ACTIVATION_CODES 是**整段替换**，不是追加。卖码时用 Excel 记录全部有效码，每次粘贴完整列表。

#### 改环境变量后必须 Redeploy

保存后右下角会出现蓝色提示：

```
Added Environment Variable successfully.
A new deployment is needed for changes to take effect.
                                              [ Redeploy ]
```

**点 Redeploy**，或去 Deployments 手动 Redeploy。否则新变量不生效。

## 5. 域名解析完整流程（重点）

本章说明：在阿里云购买域名后，如何把 DNS 解析到 Vercel，并与企业邮箱共存。

### 5.0 整体流程图

```
阿里云购买域名 shuzigongju.com
    ↓
（可选）购买企业邮箱 → 自动添加 MX 等记录
    ↓
Vercel 添加域名 → 获得需要的 DNS 记录
    ↓
阿里云 → 域名 → 解析 → 添加记录
    ↓
  ① CNAME  www → cname.vercel-dns.com
  ② A      @   → 76.76.21.21
    ↓
等待 DNS 生效（5～30 分钟）
    ↓
Vercel Domains → Refresh → 蓝勾
    ↓
访问 https://www.shuzigongju.com
```

---

### 5.1 域名基本信息

| 项目 | 值 |
|------|-----|
| 域名 | `shuzigongju.com` |
| 注册商 | 阿里云 |
| 用途 | 简历工坊网站 + 企业邮箱 |
| 买家访问地址 | `https://www.shuzigongju.com` |

### 5.2 进入 DNS 解析页面（不要进错）

#### ✅ 正确入口（网站 DNS）

```
阿里云控制台
  → 顶部搜索「域名」
  → 域名列表
  → 找到 shuzigongju.com
  → 点击「解析」或「DNS 解析」
```

URL 类似：`dns.console.aliyun.com/#/dns/domainList`

进入后应看到 **解析设置** 标签页，有「添加记录」蓝色按钮和已有记录列表。

#### ❌ 容易混淆的入口

| 入口 | 是什么 | 能不能配网站 |
|------|--------|--------------|
| 企业邮箱 → 邮箱管理 → 设置解析 | 只配邮箱 MX | ❌ 不够，要去域名解析 |
| 域名控制台 → 解析设置 | 正确入口 | ✅ |

---

### 5.3 添加第一条：www 的 CNAME 记录

#### 操作步骤

1. 点蓝色 **「添加记录」**
2. 在弹出窗口逐项填写：

| 字段 | 填什么 | 说明 |
|------|--------|------|
| **记录类型** | `CNAME` | 将域名指向另一个域名 |
| **主机记录** | `www` | 代表 www.shuzigongju.com |
| **解析请求来源** | `默认` | 不用改 |
| **TTL** | `10 分钟`（默认） | 不用改 |
| **记录值** | `cname.vercel-dns.com` | ⚠️ 不要多空格，不要加 https:// |
| **权重** | `1` | 默认即可 |
| **状态** | 开启（绿色） | — |

3. 点 **「确定」** 保存

#### 验证

解析列表中应出现一行：

```
主机记录: www    类型: CNAME    记录值: cname.vercel-dns.com
```

---

### 5.4 添加第二条：主域名 @ 的 A 记录

#### 操作步骤

1. 再点 **「添加记录」**
2. 填写：

| 字段 | 填什么 | 说明 |
|------|--------|------|
| **记录类型** | `A` | 将域名指向 IPv4 地址 |
| **主机记录** | `@` | 代表 shuzigongju.com（不带 www） |
| **解析请求来源** | `默认` | 不用改 |
| **TTL** | `10 分钟` | 不用改 |
| **记录值** | `76.76.21.21` | Vercel 官方 IP，固定值 |
| **状态** | 开启 | — |

3. 点 **「确定」** 保存

#### 为什么是 76.76.21.21？

- 这是 **Vercel 官方文档**指定的 apex 域名 A 记录 IP
- 主域名 `@` 不能用 CNAME（DNS 规范限制），所以用 A 记录指 IP
- Vercel 收到请求后，会根据你在 Domains 里的绑定，把流量转到你的项目

#### 验证

解析列表中应出现：

```
主机记录: @    类型: A    记录值: 76.76.21.21
```

---

### 5.5 邮箱记录：不要删、不要改

若购买过阿里云企业邮箱，解析列表里可能已有：

| 类型 | 主机 | 用途 |
|------|------|------|
| MX | @ | 邮件收发 |
| CNAME | mail / smtp / pop3 / imap | 邮箱服务 |
| TXT | @ | SPF 防伪造 |

**这些全部保留**，网站和邮箱共用同一域名，互不冲突：

```
@    A      76.76.21.21           ← 网站（新增）
@    MX     mx.qiye.aliyun.com    ← 邮箱（保留）
@    TXT    v=spf1 ...            ← 邮箱（保留）
www  CNAME  cname.vercel-dns.com  ← 网站（新增）
mail CNAME  ...                   ← 邮箱（保留）
```

### 5.6 DNS 生效与验证

#### 等待时间

| 情况 | 时间 |
|------|------|
| 通常 | 5～30 分钟 |
| 最长 | 24 小时（极少） |
| TTL 10 分钟 | 理论上 10 分钟后可生效 |

1. 打开 **Settings → Domains**
2. 点 `shuzigongju.com` 或 `www.shuzigongju.com` 旁的 **Refresh**
3. 红色 Invalid → 等一会再 Refresh
4. 变蓝勾 = 成功

#### 在浏览器验证

```
https://www.shuzigongju.com        ← 主站（推荐）
https://shuzigongju.com            ← 应自动 308 跳到 www
```

#### 本地命令验证（可选）

```bash
# 查 www 的 CNAME
dig www.shuzigongju.com CNAME +short

# 查主域名的 A 记录
dig shuzigongju.com A +short
# 应返回 76.76.21.21
```

### 5.7 域名解析常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| Vercel 一直红色 Invalid | DNS 未配或未生效 | 核对记录值，等 30 分钟 Refresh |
| 加了 A 但 www 不通 | 只配了 @ 没配 www | 补 CNAME www |
| 邮箱不能收信 | 误删 MX 记录 | 恢复 MX，或邮箱控制台「设置解析」 |
| 黄字 DNS Change Recommended | Vercel 优化建议 | 可忽略 |
| 国内打开很慢 | Vercel 在海外 | 见 [第 10 章](#10-国内访问说明) |

### 5.8 域名解析检查清单

## 7. 环境变量配置（激活码）

> 首次配置步骤见 [4.4 环境变量配置](#44-环境变量配置)。本章补充卖码维护细节。

### 7.1 配置入口（重要：是项目级，不是团队级）

```
❌ 错误：vercel.com/~/settings/environment-variables  （团队设置，没有项目变量）
✅ 正确：vercel.com/wuyixinyu/wuyixinyuxu/settings/environment-variables
```

路径：**项目 wuyixinyuxu → Settings → Environment Variables**

### 7.2 两个必配变量

| Key | 作用 | Sensitive 建议 | 示例 |
|-----|------|----------------|------|
| `ACTIVATION_SECRET` | 服务器签名密钥，买家不可见 | ✅ **开着** | `resume-secret-2026-Kp8mN2qR5vX9wZ3jL7hF4tY6` |
| `ACTIVATION_CODES` | 激活码白名单 | ❌ **关着**（方便查看编辑） | 见下方格式 |

### 7.3 激活码格式

```
激活码:类型
```

| 类型 | 含义 | 示例 |
|------|------|------|
| `lifetime` | 永久会员 | `XIANYU-VIP-8888:lifetime` |
| 数字 | 有效天数 | `XIANYU-30DAY-001:30` |

多个码用**英文逗号**分隔，**不要空格**：

```
XIANYU-VIP-8888:lifetime,XIANYU-30DAY-001:30,DEMO-2026-PRO:lifetime
```

### 7.4 ACTIVATION_SECRET 说明

- 至少 **16 位**，建议 **32 位**随机字符串
- **不是激活码**，买家不需要知道
- 配好后**不要随意修改**，改了所有已激活用户 token 失效
- Sensitive 保存后编辑**看不见值**，需自己记在备忘录

### 7.5 ACTIVATION_CODES 编辑注意事项

#### Sensitive 开着的问题

- 保存后编辑时 Value **显示为空**（Vercel 安全设计）
- **无法恢复**之前看不见的值
- 若用空 Value 保存，会**清空**所有激活码

#### 推荐做法

1. **Delete** 旧的 `ACTIVATION_CODES`
2. **Add Environment Variable** 重新创建
3. Value 填入全部有效码
4. **Sensitive 关着**（滑块在左、灰色）
5. Save → Redeploy
6. 再次 Edit，应能看到完整 Value

#### 卖码时必须用 Excel 记账

Vercel 编辑是**整段替换**，不是追加。每次修改需粘贴**全部有效码**：

| 激活码 | 类型 | 买家 | 状态 |
|--------|------|------|------|
| XIANYU-VIP-8888 | lifetime | 测试 | 有效 |
| XIANYU-20260705-A1 | lifetime | 张三 | 已售 |

## 9. 问题排查手册

本次部署遇到的所有问题汇总：

### ❌ 问题 1：Vercel 404，Production 仍是 Initial commit

**现象**：部署显示 Ready，但预览 404，Source 是 `Initial commit`（只有 README）

**原因**：完整代码没有 push 成功，Vercel 部署的是空仓库

**解决**：本地 `git push origin main`，确认 GitHub 上有 `src/`、`package.json`

---

### ❌ 问题 2：构建失败 `npm install exited with 1`（E401 Nexus）

**现象**：

```
npm error code E401
Unable to authenticate, need: BASIC realm="Sonatype Nexus Repository Manager"
```

**原因**：`package-lock.json` 里依赖地址指向公司内网 Nexus（`nexus.mcd.com.cn`），Vercel 无法访问

**解决**：

1. 添加 `.npmrc`：`registry=https://registry.npmjs.org/`
2. 删除 lock 和 node_modules，用公共源重新 `npm install`
3. commit 并 push 新的 `package-lock.json`

**Commit**：`fix: 使用公共 npm 源，修复 Vercel 构建失败`

---

### ❌ 问题 3：构建失败 `No Output Directory named "public" found`

**现象**：`next build` 已成功，但最后报找不到 `public` 目录

**原因**：Vercel Framework Preset 设成了 **Other**，把项目当静态站，Output Directory 默认找 `public`

**解决**：

1. Settings → Build and Deployment → Framework Preset 改为 **Next.js**
2. Output Directory **留空**
3. 添加 `vercel.json` 强制 `"framework": "nextjs"`

**Commit**：`fix: 添加 vercel.json 强制 Next.js 框架，修复 Output Directory 错误`

---

### ❌ 问题 4：`wuyixinyuxu.vercel.app` 国内打不开（ERR_CONNECTION_REFUSED）

**现象**：本地、手机流量均无法访问 `.vercel.app` 域名

**原因**：`*.vercel.app` 在国内常被 DNS 污染或防火墙拦截

**解决**：绑定自定义域名 `shuzigongju.com`（不能根治速度问题，但可提高可访问性）

---

### ❌ 问题 5：自定义域名能解析但访问极慢 / 白屏

**现象**：`www.shuzigongju.com` 有时能开，但加载极慢或白屏

**原因**：Vercel 服务器在海外，国内访问需跨境，延迟高

**可选方案**：

| 方案 | 说明 |
|------|------|
| 发货用 www 地址 | 比 apex 和 .vercel.app 更稳定 |
| Cloudflare CDN | 域名接入 Cloudflare 开代理，可能略快 |
| 迁阿里云/腾讯云 | 香港节点免备案；国内节点需备案，速度最好 |

---

### ❌ 问题 6：Domains 显示红色 Invalid Configuration

**现象**：添加域名后 shuzigongju.com / www 显示红色

**原因**：阿里云 DNS 还没配或未生效

**解决**：按 [5.2 节](#52-在阿里云添加网站解析) 添加 A + CNAME，等 5～30 分钟后点 Refresh

---

### ❌ 问题 7：Domains 黄色 DNS Change Recommended

**现象**：蓝勾 + 黄色「DNS Change Recommended」

**原因**：Vercel 建议更优 DNS 配置

**处理**：**可忽略**，不影响使用

---

### ❌ 问题 8：环境变量编辑时 Value 为空

**现象**：之前配过 ACTIVATION_CODES，编辑时看不到值

**原因**：创建时勾选了 **Sensitive**，Vercel 加密后不再显示

**解决**：Delete → 重建 → Sensitive **关着** → 填完整 Value → Save → Redeploy；用 Excel 记账

---

### ❌ 问题 9：git push 卡住无反应

**原因**：等待 GitHub 登录授权，或网络问题

**解决**：在本机终端执行 push，按提示完成 GitHub 登录；公司网络可试手机热点

---

### ❌ 问题 10：PDF 导出与预览不一致 / 导出很丑

**原因**：曾使用独立的 `formal-templates.tsx` 做 PDF，与屏幕预览模板不同

**解决**：已统一为 `ResumePreview`（scaled=false）导出，预览与 PDF 一致
